# E2E Multi-Block Test Fixtures Design

## Overview

Extend the E2E test infrastructure to generate fixtures for multiple Katana blocks and add comprehensive Cairo tests for journal decoding and proof verification.

## Scope

1. Modify E2E test to advance remote Katana through 3 blocks (0, 1, 2), saving artifacts for each
2. Create Rust CLI command to generate Cairo test fixtures from proof artifacts
3. Add Cairo unit tests for `journal_decode` function
4. Add Cairo contract integration tests using starknet foundry filesystem

## Component Details

### 1. E2E Test Modifications

**File**: `tests/e2e/run_e2e_tests.sh`

**Changes**:
- Advance remote Katana TEE RPC through 3 empty blocks (0 тЖТ 1 тЖТ 2)
- For each block, run fetch тЖТ prove тЖТ calldata pipeline
- Save artifacts to nested directory structure

**Fixture Structure**:
```
tests/fixtures/
тФЬтФАтФА block_0/
тФВ   тФЬтФАтФА attestation.json    # TEE quote response
тФВ   тФЬтФАтФА proof.json          # SP1 Groth16 proof
тФВ   тФФтФАтФА calldata.txt        # Garaga calldata
тФЬтФАтФА block_1/
тФВ   тФЬтФАтФА attestation.json
тФВ   тФЬтФАтФА proof.json
тФВ   тФФтФАтФА calldata.txt
тФФтФАтФА block_2/
    тФЬтФАтФА attestation.json
    тФЬтФАтФА proof.json
    тФФтФАтФА calldata.txt
```

**Note**: Block advancement happens on the remote Katana RPC (TEE), not local devnet. Local devnet is only for Starknet contract deployment/testing.

### 2. Rust CLI Fixture Generator

**Location**: `clients/amd_tee_registry_client/src/`

**New CLI subcommand**: `generate-cairo-fixtures` (or similar)

**Input**: `tests/fixtures/block_N/proof.json` files

**Output**: `contracts/amd_tee_registry/tests/test_journal_decode_fixtures.cairo`

**Generated content**:

**Source of truth**: AMD SDK for decoding journal structure into VerifierJournal fields.

### 3. Cairo Unit Tests for journal_decode

**Location**: `contracts/amd_tee_registry/tests/`

**Files**:
- `test_journal_decode_fixtures.cairo` - Generated by Rust CLI
- `test_journal_decode.cairo` - Hand-written test logic

**Test structure**:
```cairo
use amd_tee_registry::journal_decode::decode_verifier_journal;
use super::test_journal_decode_fixtures::{
    get_block_0_inputs, get_block_0_expected,
    get_block_1_inputs, get_block_1_expected,
    get_block_2_inputs, get_block_2_expected,
};

#[test]
fn test_decode_block_0() {
    let inputs = get_block_0_inputs();
    let expected = get_block_0_expected();
    let result = decode_verifier_journal(inputs.span());
    assert_eq!(result, expected);
}

#[test]
fn test_decode_block_1() { ... }

#[test]
fn test_decode_block_2() { ... }
```

**Requirement**: `VerifierJournal` struct needs `PartialEq` derive for `assert_eq!` comparison.

### 4. Contract Integration Tests

**Location**: `contracts/katana_tee/tests/test_verify_with_fixtures.cairo`

**Pattern**: Follow `crates/garaga/src/contracts/autogenerated/sp1_verifier_bn254/tests/test_contract.cairo`

**Test structure**:
```cairo
use snforge_std::fs::{FileTrait, read_txt};
use snforge_std::{declare, ContractClassTrait};

#[test]
#[fork(url: "ENV_RPC_URL", block_tag: latest)]
fn test_verify_block_0() {
    // Deploy KatanaTee contract
    let contract = deploy_katana_tee();

    // Load calldata from fixture
    let file = FileTrait::new("tests/fixtures/block_0/calldata.txt");
    let calldata = read_txt(@file);

    // Call verify_and_update_state
    let result = contract.verify_and_update_state(calldata.span(), ...);

    // Assert success
    assert(result.is_ok(), 'Block 0 verification failed');

    // Verify state was updated correctly
    let state = contract.get_latest_state();
    // assert block_number, state_root, block_hash match attestation
}

#[test]
#[fork(url: "ENV_RPC_URL", block_tag: latest)]
fn test_verify_block_1() { ... }

#[test]
#[fork(url: "ENV_RPC_URL", block_tag: latest)]
fn test_verify_block_2() { ... }
```

**Fork test**: Uses env RPC URL (testnet or mainnet). Garaga class hash is deployed on both networks.
